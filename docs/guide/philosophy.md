# 设计理念：替你把关的几件事

Prism 的目标只有一个：**让没有设计师的人，也能得到接近设计师水准的配色。**

你不需要懂下面这些概念。它们是 Prism 在背后替你守住的底线 —— 正因为有人（这里是算法）替你把住了关，你才可以只凭"我喜欢这个颜色"这一个判断，就得到一套靠谱的主题。这一页解释的，是"为什么你可以放心地不懂这些"。

## 一个颜色，本就意味着一整套

当你选定一个主题色时，其实已经隐含了一整套问题：它的浅色背景长什么样？悬停时该加深多少？拿来当文字时还看得清吗？在暗色界面里会不会显得脏？

这些问题都有**结构化的、可计算的答案**。Prism 做的，就是把你那一个颜色，推导成回答了全部这些问题的一套设计令牌。你只负责"挑色"，剩下的交给规则。

## 均匀，要让眼睛说了算

很多调色工具在数字上很"整齐"：明度等差、色相等分。但人眼不是按数字感知颜色的 —— 同样降 10% 明度，浅色端几乎看不出，中间调却变化明显。数字上的均匀，看起来反而不均匀。

Prism 在 [OKLCH](/guide/oklch) 这个**感知均匀**的空间里生成色阶，并按视觉间隔（而非算术间隔）排布深浅。结果就是：你拿到的色阶，看起来是均匀顺滑地由浅到深，而不是忽明忽暗。你不必理解这一点，但能直接受益。

## 你的颜色，原样保留

有些工具会先把你的颜色"洗"进算法，吐出来时已经面目全非。Prism 不这么做：你输入的精确颜色会被**逐字保留**，钉在最接近的那一档上（[色阶](/guide/scale)细说）。品牌色始终是你说了算，算法只在它周围生长出其余颜色。

```ts
import { generateTheme } from "@prism/core";

const theme = generateTheme({ primary: "#3b5bdb" });
// primary-500 就是你输入的那个颜色，一个比特都不差
theme.scales.primary.light.steps[500].hex; // '#3b5bdb'
```

## 搭配是算出来的，不是猜的

次色、中性色、语义色与主题色之间的关系，是有规律的，不该靠运气。Prism 用真正的[配色规则](/guide/harmony)推导它们 —— 比如次色取分裂互补、第三色按 +60° 规则 —— 并且每旋转一次色相，都重新对齐明度、收敛彩度，让新色和主题色像同一家族，而不是硬贴上去的补丁。

## 可读性是解出来的，不是赌的

"这个字放在这个底色上，应该够清楚吧？" —— 这是一句祈祷，不是工程。

Prism 把可读性当成**约束来求解**：以 APCA 为目标，在明度上二分搜索，直到对比度精确达标，再用 WCAG 作合规报告（[对比度](/guide/contrast)细说）。无论你挑的是什么颜色，正文、强调文字、按钮上的文字都被解到看得清。这正是简单"变亮变暗"工具给不了的保证。

::: tip 一句话
你只需要诚实地选一个你喜欢的颜色。其余那些需要专业判断的事，Prism 替你扛了。
:::
