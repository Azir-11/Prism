# 生成逻辑：一个颜色如何变成一套主题

这一页把 Prism 的内部流程完整地讲一遍 —— 当你调用 `generateTheme({ primary })` 时，从一个颜色到一整套主题，中间到底发生了什么。读完你就能明白，为什么结果是协调且可读的。

## 总览：一条流水线

```
你的颜色
   │
   ① 解析为 OKLCH（感知均匀的工作空间）
   │
   ② 补全缺失的颜色
   │     ├─ 次色 / 第三色  ← 配色规则（分裂互补、+60°）
   │     ├─ 语义色 info/success/warning/error  ← 锚点 + 向品牌微调
   │     └─ 中性色  ← 带一点品牌味的灰
   │
   ③ 为每个颜色生成一条 11 阶色阶（亮 + 暗）
   │     ├─ 固定明度阶（按视觉间隔排布）
   │     ├─ 钟形彩度包络（由你的颜色反推峰值）
   │     ├─ 微量色相扭转（防止两端发脏）
   │     └─ 把你的颜色原样钉进最近一档
   │
   ④ 求解文字色与按钮前景（APCA 二分搜索）
   │
   ⑤ 把每个色块收进色域（只降彩度，不裁通道）
   │
   ⑥ 暗色：在同一套角色合约下镜像明度阶
   │
   ⑦ 组装语义层 + 跑一遍对比度体检
   ▼
完整主题：scales / semantic / report
```

下面逐步展开。

## ① 解析为 OKLCH

任何 CSS 颜色（hex、`rgb()`、`hsl()`、`oklch()`…）先被解析成 [OKLCH](/guide/oklch) —— 一个**感知均匀**的色彩空间。之后所有计算都在这里进行，这是"等量变化看起来也等量"的前提。

```ts
import { parseColor } from "@prism/core";
parseColor("#3b82f6"); // → { l: 0.623, c: 0.214, h: 259.8 }
```

## ② 补全你没给的颜色

只给了 `primary`？那么其余基色按规则[调和](/guide/harmony)出来：

- **次色**：分裂互补，色相 `+150°`；
- **第三色**：Material 的 `+60°` 规则，彩度减半；
- **语义色**：从固定锚点出发（error≈27°、warning≈75°、success≈150°、info≈250°），向你的品牌色相**最多偏 15°**，再夹回安全区间 —— 这样既和品牌呼应，红色又仍然像"危险"；
- **中性色**：取品牌色相、极低彩度的灰，让灰也"有一点品牌的味道"。

关键铁律：每旋转一次色相，都**重新对齐明度、把彩度收敛到统一目标**，新色才会和主题色像一家人。任何一项你都可以显式覆盖。

## ③ 把每个颜色铺成一条色阶

这是引擎的核心（[色阶](/guide/scale)细说）。对每个基色：

1. **取一条固定明度阶**：50→950 共 11 档，按视觉间隔排布（中间调跳变大、两端收敛），不是等差直线。
2. **套一层钟形彩度包络**：彩度在中间调（L≈0.62）最高，向白、向黑两端衰减 —— 因为接近纯白/纯黑本就承载不了彩度。峰值由**你的颜色自身彩度反推**：闷的颜色得到闷的色阶，鲜的得到鲜的。
3. **加一点色相扭转**（默认 4°），让最深和最浅处不发灰、不发脏。
4. **把你的颜色原样钉进最近一档**。因为明度阶单调，种子总落在邻居之间，既不破坏顺序，也一个比特不改。

## ④ 求解文字色与按钮前景

文字色不靠约定，而是[求解](/guide/contrast)：以第 100 档为背景，用 `solveForLc` 在明度上二分搜索，直到 APCA 对比度命中目标 —— 正文 `Lc 60`、强调文字 `Lc 90`。按钮上的前景（`on-solid`）则在白色与深色之间，按 APCA 选更清楚的那个。

正因为是"解"出来的，无论你挑什么颜色，文字都达标。

## ⑤ 收进色域

OKLCH 里鲜艳的中间调常常超出 sRGB。Prism **固定明度与色相、只降低彩度**把它收回色域（而不是粗暴裁剪 RGB 通道 —— 那会让颜色偏色）。设成 `gamut: 'p3'` 则放宽到广色域，中间调更鲜。

## ⑥ 暗色：镜像明度阶

暗色不是反色，而是在**同一套角色合约**下把明度阶镜像过来（最浅的背景变最深，文字变最亮），并把中间调彩度提约 15%，补偿暗背景下颜色显得更暗淡的视觉效应（[暗色模式](/guide/dark-mode)细说）。组件用的语义令牌一字不改，整套就翻转了。

## ⑦ 组装语义层 + 体检

最后，Prism 把各色阶的档位映射成贴近 shadcn 的[语义令牌](/api/core)（`background`、`primary`、`primary-foreground`、`border`、`muted`…），并跑一遍对比度体检，产出 `report`：

```ts
const theme = generateTheme({ primary: "#3b82f6" });
theme.report.passes; // true：所有文字配对均达到 APCA 目标
theme.report.checks; // 每一对前景/背景的 APCA Lc 与 WCAG 比值
```

到这里，你手上就是一套 `scales`（数字色阶）+ `semantic`（语义令牌）+ `report`（体检报告）的完整主题，可直接交给 [@prism/css](/api/css) 或 [@prism/tailwind](/api/tailwind) 落地。
