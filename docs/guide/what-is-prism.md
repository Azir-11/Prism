# Prism 是什么

Prism 是一个**主题色板生成引擎**。你给它一个（或几个）种子色，它还你一整套**和谐、可访问、明暗成对**的设计令牌 —— 数字色阶、语义 token、对比度审计，以及可以直接落地的 CSS 变量、Tailwind `@theme` 或 JSON。

它的立场很明确：

- **OKLCH-first** —— 所有计算都在感知均匀的色彩空间里完成；
- **对比度是解出来的** —— 用 APCA 二分求解文字色，而不是靠经验拍脑袋；
- **核心与适配器分离** —— 引擎只管颜色，落地交给适配器。

## 它解决什么问题

手工维护一套设计系统的颜色是件苦差事：你要为每个品牌色调出 11 个深浅，要保证暗色模式不糊、文字处处可读，还要让"成功绿""警告黄"与品牌色看起来像一家人。大多数工具用 HSV 简单加深/提亮 —— 快，但既不感知均匀，也不保证对比度。

Prism 把这些都变成确定性的计算：

| 你给的                               | Prism 推导出的                           |
| ------------------------------------ | ---------------------------------------- |
| `primary`                            | 11 阶主色板 + 文字色 + on-solid 前景     |
| （可选）`secondary` / `tertiary`     | 缺省则按色彩调和规则派生                 |
| （可选）`info/success/warning/error` | 缺省则从品牌色合成、并锚定在可识别的色相 |
| （可选）`neutral`                    | 缺省则取品牌色相微染的中性灰             |
| ——                                   | 亮色 + 暗色两套，以及一份对比度报告      |

## 架构：core + adapters

```
@prism/core        颜色科学与令牌生成（唯一依赖 culori）
   ├── @prism/css        → :root { … } + .dark { … }
   └── @prism/tailwind   → @theme { … } + .dark 覆盖
```

`@prism/core` 的返回值本身就是可序列化的 JSON，因此 "JSON 输出" 几乎是免费的。适配器都是纯函数 `PrismTheme → 字符串/对象`：core 不碰任何输出格式，适配器不碰任何颜色数学。想要 shadcn token、Material 方案、或者你自家的设计令牌格式？再写一个适配器即可，引擎一行都不用动。
