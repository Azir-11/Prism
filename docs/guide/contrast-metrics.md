# 对比度指标：APCA 与 WCAG

文字看不看得清，不该靠肉眼估。Prism 用两套指标给每一对前景/背景打分：**APCA** 作为生成目标，**WCAG 2.x** 作为合规参考。这一页解释它们各自是什么、怎么打分、满分多少、达标线在哪 —— 也就是 playground 里那个圆圈问号展开后的完整说明。

## WCAG 2.x 对比度

**是什么。** WCAG 2.x 是现行的无障碍标准，也是绝大多数法规直接引用的标准。它把两种颜色的**相对亮度**算成一个比值。

**怎么打分。** 取两色的相对亮度 `L1`（亮）与 `L2`（暗），对比度为：

```
对比度 = (L1 + 0.05) / (L2 + 0.05)
```

**满分与达标线。** 范围从 `1:1`（两色相同、无对比）到 `21:1`（纯黑配纯白，**满分**）。常用门槛：

| 级别 | 正文文本  | 大号文本（≥18.66px 粗体 / ≥24px） |
| ---- | --------- | --------------------------------- |
| AA   | ≥ 4.5 : 1 | ≥ 3 : 1                           |
| AAA  | ≥ 7 : 1   | ≥ 4.5 : 1                         |

**优点**：算法简单、引用广泛、有法律效力。**短板**：它只看亮度比、不区分明暗极性，也不考虑字号字重，因此在**暗色配对**和**中间色调**上常常**高估**实际可读性 —— 一对在 WCAG 下"达标"的暗色文字，肉眼看可能仍偏弱。

## APCA（Lc）

**是什么。** APCA（Accessible Perceptual Contrast Algorithm，算法代号 SA98G）是 WCAG 3 草案采用的新一代对比度模型。它输出 **Lc（Lightness Contrast）** 值。

**怎么打分。** APCA 在估算亮度时更贴近人眼，并且**区分极性**（深字浅底 vs 浅字深底）、把字号字重纳入考量。输出范围约 `0–106`（深字浅底）/ `0–108`（浅字深底），数值越大对比越强。

**满分与参考线。** 满分 ≈ `106`。按用途的参考线：

| Lc  | 适用                       |
| --- | -------------------------- |
| 90  | 正文首选                   |
| 75  | 正文最低                   |
| 60  | 大号 / 流畅文本            |
| 45  | 大号粗体                   |
| 30  | 非正文（占位、禁用态）最低 |
| 15  | 仅装饰 / 非文本            |

**优点**：更符合主观感受，尤其在暗色模式下比 WCAG 准得多。**现状**：属于 WCAG 3 草案，尚未成为可直接援引的法规标准 —— 所以 Prism 用它来"把关质量"，同时仍报告 WCAG 供合规。

## 两者对比

|              | WCAG 2.x        | APCA（SA98G） |
| ------------ | --------------- | ------------- |
| 输出         | 比值 1:1 – 21:1 | Lc 0 – ~106   |
| 满分         | 21:1（纯黑白）  | ~106          |
| 是否分极性   | 否              | 是            |
| 暗色准确度   | 偏乐观          | 高            |
| 法规地位     | 现行标准        | WCAG 3 草案   |
| Prism 的用途 | 合规报告        | **生成目标**  |

## Prism 怎么用它们

Prism **以 APCA 为生成目标**来"解"颜色，而不是事后检查（细节见 [对比度：求解而非祈祷](/guide/contrast)）：

- 正文文字色解到 **Lc 60**，强调文字色解到 **Lc 90**；
- 按钮等实色上的前景（`on-solid`）在白色与深色间按 APCA 选更清楚的那个；
- 无论你挑什么色相，这些目标都被解到位 —— 这正是简单"变亮变暗"工具给不了的。

每次 `generateTheme` 都会产出一份体检报告：

```ts
const theme = generateTheme({ primary: "#3b82f6" });

theme.report.passes; // 是否全部达到各自的 APCA 目标
theme.report.checks[0];
// {
//   token: 'foreground on background',
//   appearance: 'light',
//   apcaLc: 98.6,      // 实际得分（带符号，正=深字浅底）
//   apcaTarget: 75,    // 这一对期望达到的 Lc
//   wcag: 16.1,        // WCAG 比值（满分 21）
//   passApca: true,
//   passWcagAA: true,  // 是否 ≥ 4.5
// }
```

也可以对任意主题随时复检：

```ts
import { verifyContrast } from "@prism/core";
const report = verifyContrast(theme);
```
